# 판단일지 (Judgment Diary) – Implementation Plan
> file: 20260215_judgment-diary.plan.md

## 0. 목적 (Intent)
- **투자 성과(수익률) 기록이 아니라, 판단(Decision) 축적**을 위한 공개 아카이브를 만든다.
- 각 기록은 “그 시점의 관찰 → 가설/판단 → 실행 → 검증”으로 재현 가능해야 한다.
- 장기적으로는 **나의 의사결정 패턴을 분석**하고, 외부 독자에게는 **사고 과정이 읽히는 콘텐츠**가 되게 한다.

## 1. 범위 (Scope)
### In scope
- 판단일지 글(엔트리) 생성/수정/조회
- 공개 페이지(리스트 + 상세)
- 태그/자산/전략 등 분류 체계
- 검증(후기/리뷰) 섹션: “결과/배운 점/다음 액션”
- 최소한의 검색/필터(기간, 자산, 태그, 포지션 변화)

### Out of scope (v1)
- 자동 매매 연동, 브로커 API 체결 자동 수집
- 수익률 리포트/백테스트 엔진
- 유료 구독/결제
- 댓글/커뮤니티(필요 시 v2)

## 2. 핵심 원칙 (Design Principles)
- **판단 중심**: “무엇을 샀나”보다 “왜 그렇게 판단했나”가 1순위
- **검증 가능성**: 나중에 맞/틀을 평가할 수 있는 “깨지는 조건(Invalidate)” 포함
- **가벼운 입력, 강한 구조**: 작성은 빠르게, 구조는 고정
- **공개 친화**: 종목 추천/유사 투자자문 오해를 줄이는 문구/면책 포함

## 3. 엔트리 데이터 모델 (Entry Schema)
각 판단일지는 아래 필드를 가진다. (v2 Light)

### Required
- `id`: string (unique)
- `created_at`: datetime (KST)
- `title`: string (짧고 명확하게)
- `summary`: string (한 줄 판단)
- `action`: enum [`BUY`, `SELL`, `HOLD`, `REBALANCE`, `WATCH`]

## 4. 작성 템플릿 (Judgment Diary v2 Light - Minimal)
AI agent는 아래 섹션을 그대로 생성한다(순서 고정).

1) 제목  
2) 한 줄 판단  
3) Action  

## 5. 공개용 면책 문구 (Default Disclaimer)
- “본 기록은 개인의 판단 과정을 정리한 것이며 투자 권유가 아닙니다.”
- “모든 투자의 책임은 본인에게 있습니다.”

## 6. 화면/UX (v2 Light)
### List Page
- 최신순 리스트
- 필터: Action
- 카드 정보: 제목, 한 줄 판단, Action

### Detail Page
- 템플릿 섹션을 그대로 렌더링
- 관련 글 추천: 추후

## 7. 운영 워크플로우 (Agent Workflow)
1) 사용자가 “오늘 판단일지 작성” 요청
2) agent는 사용자의 입력(포트폴리오 스냅샷/시장 관찰/하려는 행동)을 받아
3) 템플릿 v1에 맞춰 초안 생성
4) 사용자 확인 후 저장
5) 일정 기간 후(예: 1주/1개월) “리뷰 작성”을 유도(알림은 v2)

## 8. MVP 완료 기준 (Acceptance Criteria)
- [ ] 엔트리 CRUD 가능(최소: 생성/조회/수정)
- [ ] 리스트/상세 공개 페이지 동작
- [ ] Action 필터 동작

## 9. 리스크 & 대응
- **투자자문 오해** → 판단 기록임을 명시하는 문구 추가(필요 시)
- **과도한 입력 부담** → 필드 3개로 최소화
- **일관성 붕괴** → 템플릿 순서 고정

## 10. 다음 결정해야 할 것 (Open Questions)

- 확장 시점: 필드 추가(시장 구조/리스크/점검 등) 시점
