# 판단일지 카드 리팩토링 개발 가이드
> file: judgment-diary-card-refactor.plan.md

## 1. 목적 (Objective)
현재 판단일지 카드는 “로그 UI”에 가깝다. 이를 “Growth Artifact 카드”로 전환한다.

### 핵심 변화
- Action 중심 → 제목 중심
- 기록 느낌 → 선언 느낌
- 보고서 → 철학 카드

## 2. UX 철학
판단일지는 다음 순서로 읽혀야 한다:
1) 태도 (제목)
2) 철학 (한 줄 판단)
3) 맥락 (Action은 보조 정보)

Action은 주인공이 아니다. 제목과 한 줄 판단이 주인공이다.

## 3. 카드 구조 변경
### ❌ 현재 구조
```
WATCH
제목
요약 1
요약 2
```

### ✅ 리팩토링 구조
```
[제목]                        [Action Badge]

한 줄 판단 (강조 스타일)
```
필요 시 날짜는 매우 작게 하단에 배치.

## 4. 레이아웃 가이드
### 카드 컨테이너
- max-width: 720px
- padding: 32px
- border-radius: 16px
- background: white
- box-shadow: subtle (0 4px 12px rgba(0,0,0,0.05))

### 제목 스타일
- font-size: 20~22px
- font-weight: 600
- line-height: 1.4
- margin-bottom: 12px

제목이 가장 커야 한다.

### 한 줄 판단 스타일 (핵심)
- font-size: 16~18px
- font-weight: 400~500
- line-height: 1.6
- color: #444
- margin-top: 8px

옵션:
- 왼쪽 4px 세로 강조선
- 또는 살짝 다른 배경 (#fafafa)

### Action 배지
- 위치: 제목 오른쪽 상단
- font-size: 12px
- padding: 4px 10px
- border-radius: 999px

배지는 정보 표시용이지, 시각적 중심이 아니다.

### Action 색상
- BUY: 연한 초록
- SELL: 연한 빨강
- HOLD: 회색
- WATCH: 파랑
- REBALANCE: 보라

## 5. 제거할 요소
카드에서 제거:
- Action 텍스트를 큰 글씨로 표시하는 것
- 반복 요약
- 불필요한 메타 정보
- 구분선 남발

## 6. 읽기 경험 설계
카드를 보고 사용자가 느껴야 하는 것:
> “이 사람은 이런 태도로 시장을 본다.”

읽는 데 3초 이상 걸리면 실패.

## 7. 확장 가능성 설계
이 구조는 이후 다음 확장이 가능해야 한다:
- 태그 필터
- “내 투자 원칙 모음” 자동 페이지
- 월간 태도 아카이브
- SNS 공유 카드 변환

따라서 카드 구조는 단순해야 한다.

## 8. 접근성 고려
- 모바일 기준 16px 이상
- 텍스트 줄 길이 60~75자
- 대비 충분히 확보

## 9. 예시 (최종 모습)
```
현금이 편할 때는 대기한다                     WATCH

나는 불확실한 상황이 아니라,
내가 불안해질 때 매수하지 않는다.
```

하단 작은 텍스트:
2026.02.15

## 10. 완료 기준 (Acceptance Criteria)
- [ ] Action은 보조 정보로 위치 변경
- [ ] 제목이 카드에서 가장 시각적으로 강함
- [ ] 한 줄 판단이 강조 스타일 적용
- [ ] 카드 내 텍스트 블록 2개 이하
- [ ] 읽기 시간 3초 이내

## 11. 철학적 기준
판단일지는:
- 투자 툴이 아니다.
- 데이터 대시보드가 아니다.
- 태도 아카이브다.

UI가 이를 말해줘야 한다.
